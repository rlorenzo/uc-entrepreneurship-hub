# Program-catalog audit playbook

How to audit the program catalog: verify every listed site is alive and
accurate, purge junk, find missing programs, and ship the fixes as a PR.
A scheduled cloud routine ("Quarterly program-catalog site audit" at
claude.ai/code/routines) runs this quarterly; this doc is the canonical
playbook so a local Claude Code session — or a human — can run the same
audit on demand and the method survives changes to the routine config.

First run: July 2026 (PR #19), which fixed a dead site, a duplicate
listing, ~20 stale/wrong facts, excluded 18 junk crawled entries, and
added 26 programs.

## Where the data lives

| File                             | Role                                                                                                                                                                    |
| -------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `src/data/programs.ts`           | Curated entries — hand-maintained, win on merge. Also holds `EXCLUDED_CRAWLED_IDS`. **This is the file you edit.**                                                      |
| `src/data/programs.generated.ts` | Crawler output. **Never edit by hand** — regenerated by `vp run build-data`. Suppress bad entries via `EXCLUDED_CRAWLED_IDS`.                                           |
| `src/data/normalize.ts`          | `mergePrograms` combines the two: matched by slug, then campus+name, then canonical URL (case/hash/trailing-slash insensitive). Curated fields win; crawled fills gaps. |
| `data/crawled/*.json`            | Raw per-campus crawl output with `crawledAt` and candidates. Read-only, but a key evidence source.                                                                      |
| `scripts/crawl/sites.json`       | One crawl seed per campus. Keep in sync with `hubUrl` in `src/data/campuses.ts`.                                                                                        |

Build the merged list with `npx tsx` by importing `PROGRAMS` from
`src/data/programs.ts` and printing id/name/campus/url/deadline/funding.

## Step 0 — pipeline health (report, don't fix)

Before auditing sites, read every `data/crawled/*.json`:

- A campus whose `crawledAt` is much older than the others means the
  crawler is blocked or failing there (this is how Merced's WAF problem
  surfaced).
- A campus with **zero candidates** whose seed page plainly contains
  program links in its raw HTML is a crawler extraction bug (Berkeley
  and Riverside had this in July 2026).

Report both in the PR's "Pipeline health" section. The audit itself is
data-only — crawler fixes get their own branch.

## Per-program audit

### 1. Liveness — four distinct cases

- **WAF bot-blocking.** Hosts under `ucmerced.edu` and `ucdavis.edu`
  (and some `rady.ucsd.edu` paths) return 403 to non-browser clients.
  A 403 there is _not_ a dead page. Strongest disproof: a recent
  successful crawl of that host in `data/crawled/`. Otherwise verify
  via web search.
- **JS-only or garbled pages.** An HTTP 200 with empty or unreadable
  text is _not_ dead — the site may render client-side (freeventures.org)
  or serve odd encodings (tmp.ucsb.edu). Corroborate via search and
  label the evidence as search-only.
- **Live but repurposed.** A 200 is not enough: confirm the page still
  describes _this_ program. A redirect to a homepage usually means the
  program page is gone — find where the program lives now.
- **Actually dead.** 404/410, DNS failure (including a CNAME to a
  defunct hosting provider — how venturelab.ucmerced.edu was caught),
  or connection refused/timeout, confirmed on a second attempt. Before
  deleting a dead curated entry, search for a successor or renamed
  program; if none, remove it and flag under "Needs human review".

### 2. Fact accuracy

Compare stored `desc`, `deadline`, `funding`, `selectivity`,
`cohortSize`, `eligibility` against the live page. Replace any
already-past deadline with the verified next-cycle date or a cycle
phrase ("Annual — see program site"); never leave a past date.
Restructurings count: if a program split into tracks or changed its
funding model, rewrite the desc to match reality.

### 3. Duplicates and overlap

No two entries may share a canonical URL (ignore case, `#fragment`,
trailing slash) — `mergePrograms` now dedupes this, but check. Watch
for umbrella-vs-child overlap (a directory page listed alongside the
concrete program it describes): keep the concrete program, exclude the
directory. Awardee-announcement pages are content _about_ a program,
not programs.

### 4. Junk triage — exclude or override

Crawled entries that aren't real entrepreneurship programs (awardee
announcements, internal service/team pages, link directories, expired
funding rounds, research-grant administration, clinical collaborations)
go into `EXCLUDED_CRAWLED_IDS` with a one-line reason. Ids derive from
page titles, so a retitled page resurfaces for re-review — intended.

To **fix** a crawled entry instead of removing it (wrong name, type, or
desc), add a curated entry with the _same URL_: `mergePrograms` pairs
them by canonical URL and every curated field wins. Precedent: the
`ucsc-lean-launchpad` entry overrides the crawled "NSF I-Corps Program"
record.

### 5. Gaps

Per campus, search for significant, currently-active, campus-affiliated
programs missing from the catalog (accelerators, incubators,
competitions, student hubs, proof-of-concept funds). Verify each
candidate URL is live before adding. New entries follow the field shape
and style of the "Added by the 2026-07 catalog audit" section.

### 6. Seed quality

Ask, per campus: is the seed in `scripts/crawl/sites.json` still the
best single entry point for that campus's ecosystem, given what gap
research surfaced? If a clearly better hub exists and is live, update
the seed _and_ the campus `hubUrl` together and justify the swap in the
PR. (July 2026: Merced moved from research.ucmerced.edu — which yielded
only closed climate-grant pages — to entrepreneurship.ucmerced.edu.)

## Conventions

- **Never fabricate a number.** Every dollar amount, percentage, cohort
  size, or date comes from a fetched page or a citable search result.
  Unknowns stay "Not disclosed", "See program site", or `null`.
- `type`, `stage`, `eligibility`, `duration` come from the canonical
  lists in `src/data/types-list.ts`.
- `cohortSize` means per-cohort intake — not companies in residence,
  finalist counts, or portfolio totals (those can live in the desc).
- UC-wide programs are cataloged under their host campus (Big Ideas,
  LAUNCH → berkeley).
- Degree programs and one-off events don't fit the schema — skip them.
  University-funded but externally-run organizations (nonprofit
  operators, regional SBDCs, off-campus domains) are borderline: flag
  under "Needs human review" instead of adding.
- Descriptions: 1–2 factual, card-length sentences in the existing style.
- Never touch `data/crawled/`, `data/news/`, or generated files.

## Evidence labeling

Track how each fact was verified — fetched page (strong), recent crawl
in `data/crawled/` (strong for liveness), or search snippet only (weak;
unavoidable for WAF-blocked hosts) — and mark weak-evidence changes in
the PR so a human can spot-check them.

## Delivery

1. `vp test`, `vp lint`, `vp build` must pass. The pre-commit hook adds
   formatting, type checks, and a fallow health gate. Known quirk:
   large data churn can push an _untouched_ file over the fallow
   refactoring-target threshold. Do the suggested cleanup if it's
   actionable; if it's structural and unactionable, add a narrow,
   commented ignore in `.fallowrc.json` (see the compare.tsx precedent
   there) and call it out in the PR. Never commit with `--no-verify`.
2. Branch `fix/catalog-audit-YYYY-MM`, Conventional Commits message
   (`fix(data): …`) summarizing dead links, fact corrections,
   exclusions, and additions.
3. PR against `main` with sections: **Pipeline health**; per campus —
   dead/redirected URLs, fact corrections (old → new, with source),
   newly excluded ids with reasons, added programs with evidence
   strength; **Seed changes**; **Needs human review** for anything
   unverifiable, borderline, or unresolved.
